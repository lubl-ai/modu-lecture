# Value Proposition Sheet (VPS) v0.1 — 금태 해외 수출 서비스

작성일: 2026-06-21
버전: v0.1 (1~2주차 분석 기반 초안)

---

## 1. VPS 핵심 9개 항목

### 1.1 Customer Segment (고객 세그먼트)

**주요 세그먼트**: 일본 고급 레스토랑 식자재 조달 담당자

- 업종: 도쿄 소재 오마카세·일식 파인다이닝 (객단가 3만엔 이상)
- 역할: 식자재 구매 담당 (셰프 겸 오너 또는 전담 구매 담당자)
- 규모: 연간 수산물 구매 규모 1~3억엔
- 소싱 패턴: 현재 도매시장(도요스) 또는 기존 수산 브로커 경유
- 지리: 도쿄 중심, 오사카·나고야 확장 예정

**보조 세그먼트** (2~3년차 확장):
- 동남아 프리미엄 호텔 식자재 조달 담당 (싱가포르·방콕·하노이)
- 일본 수입 식자재 벤더/도매상 (현지 유통 파트너)

---

### 1.2 Job-to-be-Done (핵심 Job)

**핵심 기능적 Job**
> "프리미엄 어종 수산물을 안정적 품질·납기로 소싱하여 코스 메뉴 구성을 유지하고 싶다."

**감성적 Job**
> "신뢰할 수 있는 해외 공급처를 확보해 바이어로서의 전문성을 입증하고 싶다."

**사회적 Job**
> "독자적인 소싱 루트를 통해 경쟁 레스토랑과 식재료 차별화를 이루고 싶다."

---

### 1.3 Pains (고통 3개)

| # | Pain | 강도 | OS 점수 (2주차 기반) |
|---|------|------|----------------------|
| P1 | 선도 불량 — 수입 수산물이 도착 시 선도 저하, 코스 메뉴 구성 차질 | 높음 | 16 |
| P2 | 납기 불안 — 발주 후 도착일 예측 불가, 긴급 대체 소싱 비용 발생 | 높음 | 16 |
| P3 | 서류·통관 복잡성 — 위생증명서·원산지증명 등 일본어 서류 처리 부담 | 중간 | 15 |

**보조 Pain**
- 공급처 다변화 어려움: 기존 브로커 의존도 과다, 품질 편차 통제 불가
- 어종 정보 부족: 조리법·규격 정보를 영어/일본어로 제공하는 한국산 공급처 부재

---

### 1.4 Gains (기대 이익 3개)

| # | Gain | 중요도 |
|---|------|--------|
| G1 | 납기 SLA 보장 — 발주 후 도착일을 계약으로 명시, 지연 시 보상 체계 확보 | 필수 |
| G2 | 선도·품질 일관성 — 규격화된 콜드체인으로 매 납품 동일 품질 보장 | 필수 |
| G3 | 소싱 업무 효율화 — 일본어 전담 창구·서류 원스톱 처리로 행정 부담 제거 | 기대 |

**보조 Gain**
- 메뉴 스토리텔링 소재: "한국 청정 해역 양식 금태" 브랜드 내러티브 활용 가능
- 독점 소싱 경쟁우위: 한국산 금태 취급 희소성으로 레스토랑 차별화

---

### 1.5 Products & Services (제공 항목)

**핵심 제품**
- 금태 활어·선어 정기 공급 서비스 (주 1~2회 납품 사이클)
- HACCP 인증 완비 패키지 (국내 HACCP + 일본 수입 요건 충족)

**서비스 패키지**
- 납기 SLA 계약서 (도착 보장일 명시, 지연 보상 조건 포함)
- 일본어 전담 창구 (담당자 1인 고정 배정, KakaoTalk/LINE 소통)
- 통관 서류 원스톱 처리 (위생증명서, 원산지증명서, 수입신고 지원)
- 콜드체인 실시간 추적 (출하~도착 온도 로그 제공)
- 조리 가이드 (일본어판: 손질법, 추천 조리법, 규격별 용도)
- 샘플 프로그램 (초기 2kg 무료 샘플, 선도 보증)

---

### 1.6 Pain Relievers (각 Pain 해소 방법)

| Pain | Pain Reliever | 구현 방식 |
|------|---------------|-----------|
| P1: 선도 불량 | 출하 전 선도 체크·온도 이력 제공, 도착 불량 시 전량 교환 보증 | 콜드체인 IoT 모니터링 + 선도 SLA 계약 |
| P2: 납기 불안 | 납기 SLA 계약 (도착일 ±1일 보장), 실시간 물류 추적 링크 제공 | 물류 파트너 고정 + 추적 API 연동 |
| P3: 서류 복잡성 | 통관 서류 일괄 대행, 일본어 서류 사전 준비 | 전담 통관 파트너 + 일본어 담당자 |

---

### 1.7 Gain Creators (각 Gain 창출 방법)

| Gain | Gain Creator | 구현 방식 |
|------|--------------|-----------|
| G1: 납기 SLA 보장 | 계약서에 도착 보장일·지연 보상 명시 | SLA 계약 템플릿 + 물류 SLA 백업 |
| G2: 품질 일관성 | 출하 규격(중량·선도등급) 표준화, 납품마다 품질 성적서 동봉 | 양식장-선별-포장 QC 프로세스 |
| G3: 업무 효율화 | 일본어 LINE/KakaoTalk 전담 채널, 24시간 이내 응대 보장 | 전담 CS 담당자 1인 배정 |

---

### 1.8 Fit 요약 (Pain-Solution / Gain-Product 매칭)

**Pain-Solution Fit**

```
P1 선도 불량    ←→  콜드체인 IoT + 선도 SLA + 도착 불량 교환 보증
P2 납기 불안    ←→  납기 SLA 계약 + 실시간 물류 추적
P3 서류 복잡성  ←→  통관 서류 원스톱 + 일본어 전담 창구
```

**Gain-Product Fit**

```
G1 납기 SLA    ←→  SLA 계약서 + 물류 파트너 고정
G2 품질 일관성  ←→  HACCP 완비 + QC 표준화 + 품질 성적서
G3 업무 효율화  ←→  일본어 담당자 + LINE 전담 채널
```

---

### 1.9 Success Metrics (측정 지표 3개)

| 지표 | 목표값 (1년차) | 측정 방법 |
|------|---------------|-----------|
| 납기 준수율 | 95% 이상 (±1일 기준) | 납품 이력 로그 |
| 선도 불량 반품률 | 2% 미만 | 클레임 접수 건수 |
| 바이어 재주문율 | 70% 이상 (3개월 이내) | CRM 주문 데이터 |

**보조 지표**
- 신규 바이어 획득: 1년차 10개 레스토랑
- 평균 응대 시간: 24시간 이내 (LINE/카카오 메시지)
- 서류 처리 리드타임: 발주 후 48시간 이내 완료

## 2. Job-MVP Feature Map

### 매핑 기준

- OS 점수: 2주차 AOS/DOS 분석 결과 (OS = Importance + (Importance - Satisfaction))
- 우선순위: Must (MVP 필수) / Should (2차 릴리즈) / Could (3차 이상)

### Job-Feature 매핑 표

| JTBD Job | 관련 Feature | 우선순위 | 근거 (OS 점수) |
|----------|--------------|----------|----------------|
| 납기 도착일을 계약으로 확정하고 싶다 | 납기 SLA 계약서 템플릿 + 지연 보상 조건 | Must | OS=16 (납기 지연 최소화) |
| 수산물 선도를 도착 전에 확인하고 싶다 | 콜드체인 IoT 온도 로그 + 도착 선도 성적서 | Must | OS=16 (선도 불량 최소화) |
| 통관 서류를 직접 처리하지 않고 싶다 | 통관 서류 원스톱 대행 + 일본어 전담 창구 | Must | OS=15 (서류 처리 시간 단축) |
| 수산물이 이동 중 어디 있는지 알고 싶다 | 물류 실시간 추적 링크 제공 (배송 API 연동) | Must | OS=14 (콜드체인 추적) |
| 초기 품질을 리스크 없이 검증하고 싶다 | 무료 샘플 프로그램 (2kg, 선도 보증) | Should | 바이어 CJM 평가 단계 Pain |
| 금태 조리법을 일본어로 이해하고 싶다 | 일본어 조리 가이드 (PDF + 규격별 용도 포함) | Should | 페르소나 온보딩 니즈 |
| 정기적으로 안정 수량을 소싱하고 싶다 | 정기 발주 계약 (주 1~2회 납품 사이클) | Must | 핵심 수익 모델 |
| HACCP 인증 여부를 서면으로 확인하고 싶다 | HACCP 인증서 + 위생증명서 사전 제공 | Must | OS 검증 핵심 성패 변수 |
