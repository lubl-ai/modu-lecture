## 1. 제품 개요

### 1.1 목적 및 범위

**목적**
일본 고급 레스토랑 식자재 바이어가 겪는 수입 수산물의 선도 불량·납기 불안·서류 복잡성 문제를 해소하고, HACCP 완비 + 납기 SLA + 콜드체인 + 일본어 전담 창구 패키지로 한국산 금태를 안정 공급하는 B2B 수출 서비스.

**범위 (In Scope)**
- 금태 활어·선어 정기 공급 (주 1~2회 납품 사이클)
- HACCP 인증 완비 및 일본 수입 요건 충족 서류 패키지
- 냉장 콜드체인 전 구간 운영 + IoT 온도 모니터링
- 납기 SLA 계약 관리 및 실시간 물류 추적
- 통관 서류 원스톱 대행
- 일본어 전담 고객 창구 (LINE/KakaoTalk)

**MVP 초기 타겟**
- 지역: 일본 도쿄 오마카세·파인다이닝 레스토랑
- 1년차 목표: 50톤, 계약 레스토랑 10개 이상, 매출 $108만~$150만

## 3. 기능 요구사항 (OS 점수 반영)

### OS 점수 기준

- OS = Importance + (Importance – Satisfaction) (2주차 AOS/DOS 분석)
- OS 15 이상: MVP Must 우선 처리
- OS 14: Must 포함 (핵심 Pain 직결)
- OS 없음: 서비스 운영 상 필수 / 전략적 필요

## 4. 기술 스택 제약 조건

### 4.1 서버: 클라우드 (AWS 또는 GCP)

| 항목 | 선택 | 이유 |
|------|------|------|
| 클라우드 공급자 | AWS (1순위) / GCP (2순위) | 한국~일본 리전 지원, 수산물 이력 데이터 GDPR/개인정보보호법 대응 용이 |
| 서버 구성 | 관리형 컨테이너 (ECS or Cloud Run) | 소규모 팀에서 인프라 관리 최소화 |
| 가용성 | 99.9% SLA 이상 필요 | 납기 추적·발주 시스템 다운 시 바이어 신뢰 훼손 |

**제약 사항**
- 자체 서버 운영 불가 (초기 CAPEX 예산 없음)
- 서버 관리 전담 인력 없음 → 관리형 서비스 필수

---

### 4.2 DB: 수산물·출하 이력 관리용 RDB

| 항목 | 선택 | 이유 |
|------|------|------|
| DB 종류 | PostgreSQL (RDS / Cloud SQL) | 발주·납품·SLA 이행·클레임 등 관계형 데이터 중심 |
| 데이터 보존 | 5년 (수출신고 법정 보존 기간) | 규정 준수 NFR-C4 |
| 백업 | 일 1회 자동 스냅샷 | 납품 이력 소실 방지 |

**주요 엔티티**
- Order (발주): 바이어ID, 발주일, 품목, 수량, 약정 도착일
- Shipment (납품): 발주ID, 출하일, 도착일, 온도 로그 링크, SLA 준수 여부
- QualityReport (품질 성적서): 납품ID, 중량, 선도등급, HACCP 로트번호
- Claim (클레임): 납품ID, 접수일, 사유, 처리 상태, 처리 완료일

---

### 4.3 IoT: 수온·선도 모니터링

| 항목 | 선택 | 이유 |
|------|------|------|
| 통신 프로토콜 | MQTT (1순위) / HTTP Polling (대안) | MQTT: 저전력 소형 IoT 기기에 적합, 실시간 푸시 가능 |
| 센서 데이터 | 온도, 습도, 위치 (GPS) | 콜드체인 이탈 즉시 감지 |
| 데이터 수집 | AWS IoT Core / GCP IoT | 관리형 MQTT 브로커, 서버리스 처리 가능 |
| 알림 | 이탈 즉시 담당자 LINE/SMS 알림 | FR-003 온도 이탈 1% 미만 달성 전제 |

**제약 사항**
- 물류 파트너사 IoT 장비 설치 협조 필요 (계약 조건 명시)
- 해상 운송 구간 GPS 수신 불안정 → 항구 환적 단계별 수동 체크 병행

## 6. Definition of Done (DoD) 체크리스트

### FR-001: HACCP 인증 완비

- [ ] 국내 HACCP 인증서 유효 기간 내 유지 확인
- [ ] 납품별 HACCP 로트번호 성적서 동봉 프로세스 수립
- [ ] 일본 통관 기준 위생증명서 양식 확정 및 템플릿화
- [ ] 일본 통관 거부 테스트 케이스 0건 확인 (초도 납품 전)

### FR-002: 납기 SLA 계약 관리

- [ ] SLA 계약서 템플릿 법률 검토 완료
- [ ] 지연 보상 조건 (금액 또는 재납품) 계약서 명시
- [ ] 납품별 SLA 이행 현황 기록 DB 구축
- [ ] 납기 준수율 95% 달성 운영 3개월 연속 확인

### FR-003: 냉장 콜드체인 운영

- [ ] 물류 파트너사와 0~5°C 유지 SLA 계약 체결
- [ ] IoT 온도 센서 전 운송 구간 설치 완료
- [ ] 온도 이탈 알림 (LINE/SMS) 자동 발송 테스트 완료
- [ ] 온도 로그 DB 기록 100% 검증

### FR-004: 실시간 물류 추적

- [ ] 물류 추적 API 연동 완료 (DHL/FedEx/EMS 중 택일)
- [ ] 발주 확정 시 바이어에게 추적 링크 자동 발송 테스트 완료
- [ ] 상태 갱신 30분 이내 반영 검증 (3회 이상 테스트)

### FR-005: 통관 서류 원스톱 대행

- [ ] 일본 통관 에이전트 계약 체결
- [ ] 서류 준비 체크리스트 확정 (위생증명서, 원산지증명서, 수출신고필증 등)
- [ ] 서류 처리 리드타임 48시간 이내 달성 검증 (샘플 납품 기준)
- [ ] 바이어 직접 처리 서류 0건 확인

### FR-006: 일본어 전담 고객 창구

- [ ] 전담 담당자 1인 채용 및 일본어 역량 검증 완료
- [ ] LINE 비즈니스 채널 바이어별 개설 완료
- [ ] 24시간 응답 SOP 문서화 및 담당자 교육 완료
- [ ] CS 만족도 조사 도구 설정 완료

### FR-007: 도착 불량 교환 보증

- [ ] 선도 등급 기준 문서화 (기준 미달 판정 기준 명시)
- [ ] 클레임 접수 → 처리 48시간 이내 SOP 수립
- [ ] 반품·교환 물류 프로세스 확정
- [ ] 클레임 처리 이력 DB 기록 체계 구축

### FR-008: 무료 샘플 프로그램

- [ ] 샘플 발송 대상 바이어 리스트 5개 이상 확보
- [ ] 샘플 콜드체인 적용 발송 절차 수립
- [ ] 피드백 수집 설문 (일본어) 작성 완료
- [ ] 샘플 발송 후 2주 이내 피드백 수집률 100% 달성

### FR-009: 출하 규격 표준화 및 품질 성적서

- [ ] 중량 규격 기준 (g 단위, 허용 편차 ±5%) 양식장과 합의
- [ ] 품질 성적서 양식 확정 및 인쇄/전자 발송 프로세스 수립
- [ ] 납품 중량 편차 ±5% 이내 3회 연속 달성 확인

### FR-010: 발주-납품 이력 기록 관리

- [ ] DB 스키마 (Order, Shipment, QualityReport, Claim) 확정
- [ ] 발주 입력 → 납품 완료 이력 자동 기록 기능 구현
- [ ] 월간 리포트 자동 생성 기능 테스트 완료
- [ ] 납품 이력 3영업일 이내 제공 프로세스 수립
